1. What is NPM

    NPM stands fro Node Package Manager. NPM is a node manager for the Node Javascript platform. Most commonly, it is used to publish, discover, install , develop and uninstall node programs and dependencies that are required to run the application NPM gets installed with NodeJs installation. NPM uses nested dependencies, so we can use different versions of any module in our code. NPM generates a 'package.lock-json' file which will generate the same node_modules folder for different npm versions.

  2. What are modules in NodeJS

    in NodeJs, modules are the blocks of encapsulated code that communicate with an external application on the basis of their related functionality. Modules can be a single file or a collection of files. The reason, modules are used is because of their ability to break down complex piece of code into smaller pieces of code to make it more manageable. There are three types of modules.

    • Internal Modules - Internal Modules are in-built modules that are predefined in NodeJS and comes with NodeJS installation. These modules can be loaded into the program by using the require function. The require() function will return a JavaScript type depending on what the particular module returns for eg. express creates an express server in NodeJS.
    • Custom Modules - Also called External Modules, these modules are locally creates in Nodejs application. For eg. if we create a calculation module that calulcates various operations, another file can use its exported functionality using the require() function.
    • Third Party Modules - These modules are available online using the NPM. These modules can be installed in the project folder or globally. Some of the commong third party modules are mongoose, express, angular and react.

  3. What is the purpose of module.exports ?

    Module.exports is the code that tells NodeJs which bits of code(functions, objects, strings etc) to export from a given file so that other files can access the exported code. The module.exports is actually a property of the module object in the node.js module. Exports is the object that is returned to the require() call.

  4. Difference between default export and names export

    Default exports are used to export only a single object, function or variable. For the export, we can use any name to import. It is mandatory to mention the "default" word in default exports. Exports without the defaul tag are named exports. Named exports are used to export several values. For the import, it is mandatory to use the same name of the corresponding object. Named exports allows us to share multiple objects, functions or variables from a single file and were introduced with the release of ES2015. 

                    function defaultExport((req, res){
                    res.send("default exports")
                })

                function namedExport((req,res){
                    res.send('named exports')
                })

                module.exports = defaultExport // default export
                module.exports = {namedExport} // named export
